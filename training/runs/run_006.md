## Why Feature Engineering

The win condition requires:
1. A horse aligned on center axis (col=6 or row=6)
2. A clear path from that horse to center (6,6)
3. A friendly backstop piece at (6,5), (6,7), (5,6), or (7,6)

The 41 raw features give the model normalised (col, row) pairs — it cannot infer axis alignment,
path clarity, or backstop occupation from coordinates alone without enormous data and capacity.

---

## New Feature Layout (105 total)

**Per horse × 20 horses = 100 features:**

| Index offset | Feature | Description |
|-------------|---------|-------------|
| +0 | `col_norm` | (col - 1) / 10 — existing |
| +1 | `row_norm` | (row - 1) / 10 — existing |
| +2 | `dist_to_center` | Manhattan distance to (6,6) / 10 — **new** |
| +3 | `on_axis` | 1.0 if col=6 or row=6, else 0.0 — **new** |
| +4 | `path_clear` | 1.0 if on-axis AND no piece between horse and center — **new** |

**Global × 5 features:**

| Index | Feature | Description |
|-------|---------|-------------|
| 100 | `backstop_(6,5)` | +1=current player, -1=opponent, 0=empty — **new** |
| 101 | `backstop_(6,7)` | same — **new** |
| 102 | `backstop_(5,6)` | same — **new** |
| 103 | `backstop_(7,6)` | same — **new** |
| 104 | `player_indicator` | 0.0=P1, 1.0=P2 — existing |
