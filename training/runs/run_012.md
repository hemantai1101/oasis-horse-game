# Run 012 — TBD

**Goal:** Recover Run 010's calibration quality while scaling up data volume.
Run 011 proved that longer games (3s time limit) produce noisier labels and hurt
leaf evaluation quality. Run 012 returns to 2s time limit to keep games short and
decisive, while increasing from 15k to 30k games and training for 150 epochs.

---

## What Changed vs Run 011

| What | Run 011 | Run 012 | Why |
|------|---------|---------|-----|
| Games | 50k | **50k** | Same volume, fix the quality |
| Time limit | 3.0s | **2.0s** | Shorter games → less label noise → better calibration |
| Records/game | ~239 | **~204** | Back to Run 010 game length |
| Epochs | 150 | **150** | Same — keep full convergence |

---

## Hypothesis

Run 010 (15k games, 2s) achieved 0.569 val loss and excellent in-game calibration.
Run 011 (50k games, 3s) achieved 0.641 val loss and poor in-game calibration.

The variable that changed was time limit (2s → 3s), which increased game length
(204 → 239 records/game). Longer games = more ambiguous mid-game positions =
noisier labels = overconfident model.

Run 012 tests whether **2s time limit with 50k games** (3× Run 010's data, same
game quality) breaks through Run 010's val loss while preserving calibration.

Expected outcome: val loss below 0.56910, d3/d4 swing < ±1,528.
