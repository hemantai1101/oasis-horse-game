# Run 010 — 2026-03-21

**Goal:** Validate whether training with depth=5 game data improves the NN's leaf
evaluations inside the browser's depth=4 minimax search. Run 009 showed the AI
defends well but doesn't attack — console logs revealed a persistent sign flip
between depth=3 (+5000) and depth=4 (-7000), meaning the NN overestimates positions
at depth=3 leaves. Hypothesis: training on stronger-play data calibrates leaf
evaluations better.

---

## What Changed vs Run 009

| What | Run 009 | Run 010 | Why |
|------|---------|---------|-----|
| Teacher depth | depth=4 | **depth=5** | Stronger play → better training labels |
| Time limit | 0.5s | **2.0s** | depth=5 needs more time to complete search |
| Games | 50k | 15k (validation run) | Test concept before committing to 50k |
| Epochs | 200 | 100 | 15k data converges faster |
| Data file | v4 clean 50k | v4 depth=5 15k | New generation |

**Skip rate note:** 20% skip rate (vs <1% for depth=4) because depth=5 play is
stronger on both sides — games that don't resolve within MAX_MOVES are skipped.
Surviving games are biased toward decisive outcomes, but quality of each game is
higher. Despite this, the training result validated the hypothesis strongly.

## Analysis

The depth=3 vs depth=4 sign flip observed in Run 009 browser logs:
```
depth=3 best=+5811    depth=4 best=-7255   ← 13000 point swing
```
This indicates NN leaf evaluations trained on depth=4 data are too optimistic.
When the browser uses them as depth=4 leaf evaluators, the search finds opponent
responses that the NN didn't account for.

Training on depth=5 data means the teacher played better, so the NN learns to
evaluate leaf positions more pessimistically/accurately — it "knows" what a
depth=5 opponent would do next. The result is calibrated leaf evaluations for
the browser's depth=4 search.
