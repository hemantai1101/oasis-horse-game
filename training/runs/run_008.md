# Run 008 — 2026-03-19

**Goal:** Fix the Run 007 regression. Keep all 5 global threat/phase features but
revert the per-horse feature back to binary `path_clear` and correct the normalization
of the global features so they produce meaningful gradient signal.

**What changed vs Run 007:**

| What | Run 007 | Run 008 | Why |
|------|---------|---------|-----|

## Root Cause of Run 007 Regression

**1. Graded `path_threat` replaced a working binary feature**

Run 006 proved that `path_clear ∈ {0.0, 1.0}` works well. The MLP can learn
`on_axis AND path_clear` because both are binary — sharp decision boundaries.
Replacing it with a continuous scale required the model to implicitly find a
threshold from MSE loss alone. This is harder and broke the clean conjunction
structure that Run 006 relied on.

**2. `/10` normalization made global features invisible**

`winning_threats` is almost always 0, 1, or 2. With `/10` normalization the
entire useful range was compressed into [0, 0.2] — nearly zero output from the
first linear layer. The model effectively ignored these features. Normalizing by
`/2` spreads the range to {0.0, 0.5, 1.0}, giving real gradient signal.

## Training Results — Key Observations

| | 15k run | 50k run |
|--|---------|---------|
| Best val loss | 0.60674 | 0.62827 |
| Train/val gap | 0.022 | 0.006 |
| Converged? | Yes (~ep 96) | No — still descending at ep 100 |
| vs Run 006 | ✅ +0.0158 better | ❌ 0.00573 worse |

**Why 15k beat 50k at 100 epochs:**
The 50k model's tiny train/val gap (0.006) shows it barely overfits — the model
simply hadn't converged yet. 100 epochs is insufficient for 5M records with
cosine LR annealing over 100 epochs. The 15k model appears better numerically
but may have partially memorised its smaller dataset distribution.

**Why this matters:** Val loss is measured on a random 5% split of the training
data. With 15k games, the val set (~75k records) comes from the same narrow
distribution as training. With 50k games, the val set (~250k records) is 3×
more diverse. The 50k model generalises to a wider position distribution —
it may actually play better despite the higher val loss number.

**Conclusion:** 100 epochs is too few for 50k v4 data. The correct run is
Run 009: same data, 200 epochs.

## Analysis — Normalization Fix Confirmed

The 15k result (0.60674) decisively confirmed the normalization fix works:
- 15k v4 data beat Run 006 (50k v2) by 0.0158
- Run 007 (50k v3, bad normalizations) was 0.63295
- The feature fix was real — the regression was the normalization, not the features themselves
