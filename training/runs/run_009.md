**Goal:** Properly converge the Run 008 50k v4 dataset. Run 008 showed the
normalization fix works (15k → 0.60674) but 100 epochs was insufficient for

## Why 200 Epochs

Run 008 (50k, 100 epochs) observations:
- Train/val gap was only **0.006** at epoch 100 — almost no overfitting
- Val loss still descending at epoch 100 (0.62849 → 0.62827)
- Compare to 15k run which converged cleanly at epoch ~96 with gap of 0.022

The cosine LR schedule anneals from 1e-3 to ~0 over `T_max` epochs. With 5M
records, each epoch does ~3.3× more gradient steps than a 1.5M record run.
Setting `T_max=200` gives the optimizer a proper warmup-to-cooldown arc across
the full dataset size.

**Expected:** val loss should continue descending from 0.628 and converge
somewhere in the 0.606–0.620 range by epoch 180–200.
