## Why 200 Epochs

Run 008 (50k, 100 epochs) observations:
- Train/val gap was only **0.006** at epoch 100 — almost no overfitting
- Val loss still descending at epoch 100 (0.62849 → 0.62827)
- Compare to 15k run which converged cleanly at epoch ~96 with gap of 0.022

The cosine LR schedule anneals from 1e-3 to ~0 over `T_max` epochs. With 5M
records, each epoch does ~3.3× more gradient steps than a 1.5M record run.
Setting `T_max=200` gives the optimizer a proper warmup-to-cooldown arc across
the full dataset size.

**Expected:** val loss should continue descending from 0.628 and converge
somewhere in the 0.606–0.620 range by epoch 180–200.

## Training Results

| Metric | Value |
|--------|-------|
| Best val loss | **0.62504** (epoch 194) |
| Final train loss | 0.61901 |
| Final val loss | 0.62571 |
| Train/val gap | 0.007 |
| Total time | 10,987s (~3.1 hrs) |
| Converged by | Epoch ~185 |

Converged cleanly. 200 epochs helped vs 100 epochs (0.62827 → 0.62504), but still
**did not beat Run 006** (0.62254). The 50k v4 depth=4 dataset appears to have a
ceiling around 0.624–0.625 regardless of epoch count.

Notably, the 15k v4 training (Run 008) achieved 0.60674 — but that is evaluated on
a smaller, less diverse val set, making the comparison misleading. The 50k val set
is harder to score well on.

---

## Outcome

Run 009 confirmed that 200 epochs is sufficient for 50k records (converged ~epoch
185). However, the fundamental ceiling for depth=4 training data was reached. The
real improvement came in Run 010 with depth=5 data.
